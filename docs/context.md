# 背景
1.现在大数据团队，针对自媒体多平台投流，多渠道采集相关数据，做撞库归因，评估不同平台不同玩法的投放效果。
2.当前存在多套代码，不同代码间的结果数据会有差异。
3.对外的数据输出有2类场景，一类是业务委托数仓做数据分析和探查，需要调整一些代码逻辑，然后取数分析，给到业务方决策。一类是日常需求交付。 
4.目前出现多次数据质量问题，给出的数据在业务做灵活的多维度分析下，部分维度组合下，出现明显的数据违背业务尝试，或者出现和其他相同指标的一致性结果差异。
5.短时间现有数仓很难快速掌握所有的业务细节，并且自测验数时，覆盖所有维度组合下的结果。这样就导致上述2类场景下， 给出的数据质量难以100%满足业务诉求
6.现在已有DQC 工具，可以部署监控规则

# 方向
1.能否借助AI，将不同数仓开发，在不同表下积累的数据自测规则和知识沉淀下来。 
--针对每个表，构建一个知识文档，后续这个表在对外取数分析，能根据知识库生成对应验数SQL； 
--日常迭代交付时，能够生成DQC 规则，保证迭代时，DQC规则能否覆盖重要核心检查项，及时发现数据异常问题。
2.随着时间流转，这个知识库由数仓/PM维护，淘汰旧的数据自测规则和知识，补充更新的数据自测规则和知识

# 目标
1.通过这个工具，能够保证不管是日常取数分析，还是需求迭代，数据质量都能保证交付质量，及时发现数据异常

# 功能
1.在表和指标粒度下的验数规则和知识，以及生成对应验数SQL，DQC规则
2.支持数仓开发、PM 维护对应验数规则和知识
3.全栈式的前后端一体，先跑通功能。 未来再考虑和已有数据平台整合。

# 不做什么
1.不做DQC 功能
2.不做复杂的平台功能


# 怎样算成功
1.能够跑通几个这样的场景
--数仓收到业务数据取数和分析需求，开始调整SQL和取数。本工具利用AI，基于知识库的验数规则和知识，生成验数SQL，本工具执行SQL并对数据做分析验证，告知数仓开发有什么问题和异常。
--数仓收到业务数据需求，完成方案设计，开始编码。本工具利用AI，基于知识库的验数规则和知识，生成DQC验数规则，本工具调用DQC 接口，执行DQC校验，并基于校验数据，告知数仓开发有什么问题和异常。
